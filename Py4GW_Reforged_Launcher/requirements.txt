# Dependency setup for the launcher's dev venv.
#
# Create the venv at .venv INSIDE this folder (Py4GW_Reforged_Launcher\.venv) --
# that exact location is the convention Launch.bat depends on:
#     py -3.13-32 -m venv .venv
#
# Then install wheels-only (plain `pip install` falls back to source builds for
# some of these on 32-bit CPython 3.13 / Windows and fails without a C compiler):
#   1. imgui-bundle 1.92.801 has no win32 wheel on PyPI, so install its prebuilt
#      wheel from this repo's standalone "wheels" release FIRST:
#        https://github.com/Royel-Payne/Py4GW_Reforged/releases/download/wheels/imgui_bundle-1.92.801-cp313-cp313-win32.whl
#      That satisfies the imgui-bundle pin below, so step 2 won't try -- and fail
#      -- to fetch it from PyPI:
#        .venv\Scripts\pip install --only-binary=:all: imgui_bundle-1.92.801-cp313-cp313-win32.whl
#   2. Then the rest:
#        .venv\Scripts\pip install --only-binary=:all: -r requirements.txt
#
# One more exception (Phase A, pywebview shell): pywebview's own dependency
# `proxy_tools` only ships an sdist, no wheel -- but it's pure Python (no C
# extension, no compiler needed), so it's safe to install from source. Add
# `--no-binary=proxy_tools` to the command above, or just drop
# --only-binary=:all: for this one package if installing it standalone.
